PROJECT SUMMARY
ResQ is an indoor and city-wide crisis-response demo platform. It helps hotels, campuses, hospitals, malls, large buildings, and smart-city command teams detect emergencies, verify incidents, coordinate responders, and guide people to safety.

The project combines a React frontend, Express backend, real-time style emergency dashboards, editable indoor floor maps, simulated people locations, hazard simulation, safest-route calculation, CCTV/YOLO demo workflows, Twilio-style alert setup, and map-assisted response tools.

Core idea:
CCTV sees. YOLO interprets. Users report. The system verifies. Staff respond. Emergency services receive cleaner incident information.

How ResQ Works:
1. Upload the floor plan
Start from DWG, DXF, SVG, or image plans. Admins can keep editing nodes after import.

2. Mark building nodes
Rooms, junctions, stairs, exits, cameras, sensors, BLE beacons, QR checkpoints, and actuators become one editable graph.

3. Detect or simulate danger
Camera and sensor events map hazards to the nearest node and block unsafe paths.

4. Guide people out
BLE estimates and QR confirmations feed safest-route guidance for guests, staff, and responders.

FLOOR MAP AND CAD DETAILS
ResQ supports floor plans and building maps.

Accepted upload types:
DWG
DXF
SVG
PNG
JPG
JPEG

Best floor map requirements:
One floor per DWG/DXF file.
Use model-space geometry, not a paper layout screenshot.
Real-world scale is preferred.
Separate layers are preferred for walls, rooms, doors, stairs, exits, fire equipment, cameras, sensors, beacons, and QR points.
Avoid exploded title blocks, unrelated furniture, heavy hatches, and duplicate routing lines.

DXF parsing support:
Lines
Lightweight polylines
Legacy polylines
Circles
Arcs
TEXT labels
MTEXT labels

DXF import behavior:
Closed room polygons on room-like layers can become editable room nodes at the room center.
Labels inside room polygons can become node names.
Corridor, path, hallway, exit, stair, camera, sensor, BLE, QR, actuator, and extinguisher layers/text can create node candidates.

DWG behavior:
DWG files are accepted and stored.
They may be marked as needing a converter unless the deployment configures a DWG conversion tool.
Admins can still place and edit nodes manually after upload.

EDGE AND ROUTING CONCEPTS
Graph nodes represent rooms, exits, devices, corridors, and checkpoints.
Graph edges connect nodes.
Edges can be open or blocked.
Hazards can block unsafe nodes and edges.
The route engine evaluates available exits and calculates the safest route.
If a person has BLE or QR location data, the system can resolve their current start node.
Route results include status, node path, selected exit, total distance, blocked nodes, blocked edges, step-by-step instructions, and a user-facing message.

SAMPLE CHATBOT RESPONSES
User: What is ResQ?
Assistant: ResQ is an emergency response platform for buildings and city operations. It combines CCTV AI, sensors, floor maps, people locations, alerts, and route calculation to help teams detect danger and guide people to safety.

User: Who is ResQ for?
Assistant: ResQ is designed for hotels, campuses, hospitals, malls, large facilities, smart cities, security teams, fire teams, medical teams, and disaster response teams.

User: How does ResQ detect danger?
Assistant: ResQ can use CCTV/YOLO detection, sensor alarms, manual reports, and simulated hazards. The system maps the danger to a floor node, marks unsafe areas, and helps calculate safer routes.

User: What happens after a hazard is detected?
Assistant: The platform verifies the incident, updates dashboards, blocks unsafe paths, notifies staff, and can guide users or responders toward safer exits or response actions.

User: What dashboards are available?
Assistant: ResQ includes local building dashboards for admins, staff, and users, plus city dashboards for fire detection, intrusion detection, medical emergency, disaster management, smart CCTV monitoring, and unified command.

User: What can the admin dashboard do?
Assistant: The admin dashboard can upload a floor map, edit nodes, connect edges, add hazards, simulate people locations, generate paths or cameras, and calculate evacuation routes.

User: What can staff see?
Assistant: Staff can monitor active hazards, people locations, route status, exits, and response actions so they can coordinate during an emergency.

User: What can users see?
Assistant: Users can select their simulated identity, confirm QR checkpoints, and view step-by-step evacuation instructions.

User: What is the fire dashboard?
Assistant: The fire dashboard monitors fire and smoke risk using a live feed, AI confidence, visibility, people count, hazard warnings, evacuation controls, and model-based detection.

User: What is the intrusion dashboard?
Assistant: The intrusion dashboard monitors suspicious activity and person detection using a live feed, threat indicators, people count, incidents, lockdown action, and AI model results.

User: What files can I upload for a floor plan?
Assistant: ResQ accepts DWG, DXF, SVG, PNG, JPG, and JPEG files. DXF files are parsed most deeply for lines, shapes, labels, and possible room/node inference.

User: Does ResQ need a database?
Assistant: The demo can run with in-memory state for easy presentations. The project also includes a Postgres-ready schema for production persistence.

User: Can ResQ use CCTV?
Assistant: Yes. It is designed to connect existing CCTV or camera feeds, run AI detection near the site, and send only event data to the backend.

User: Is this production-ready for real emergencies?
Assistant: It is a demo foundation. Before safety-critical use, models, routing, sensors, notifications, permissions, and deployment must be validated with real building layouts and real camera conditions.

User: What makes ResQ different from a normal CCTV system?
Assistant: Normal CCTV shows video. ResQ combines detection, verification, routing, alerts, people location, and responder coordination in one incident workflow.

User: How does ResQ reduce false alarms?
Assistant: It cross-checks multiple signals such as AI confidence, sensors, and location consistency. Unverified reports can be flagged instead of immediately treated as full emergencies.

User: What is the Edge AI contract?
Assistant: Edge AI should send event data such as camera ID, hazard type, confidence, timestamp, bounding box, and nearest node ID rather than sending raw video to the backend.

User: What is the safest-route engine?
Assistant: It evaluates graph nodes and edges, blocks areas affected by hazards, checks available exits, and returns a step-by-step route to the safest reachable exit.

User: What does BLE/QR support do?
Assistant: BLE gives approximate location near beacons, while QR checkpoints can confirm a person's location. These signals help the route engine start guidance from the right place.

User: What is the unified command center?
Assistant: It is a city-wide command dashboard concept that gives responders one central view across incidents, teams, alerts, and operational status.

CHATBOT BEHAVIOR GUIDELINES
Answer in a calm, practical, safety-focused tone.
Keep responses concise unless the user asks for detail.
Explain ResQ as a demo emergency intelligence and response platform.
Do not claim the system is certified for real-life emergency use.
Do not provide or reveal credentials, tokens, keys, or private configuration.
When asked about setup, mention install, dev server, frontend URL, backend URL, and database-ready design without exposing secrets.
When asked about real deployment, recommend validation, security review, model testing, incident playbooks, and integration with official emergency services.

IMPORTANT LIMITATIONS
This project is a demo and prototype-style implementation.
In-memory state is suitable for presentations.
Database schema exists for production persistence, but production storage logic must be completed and hardened.
AI detection should be retrained and validated for the actual deployment environment.
Emergency routing should be reviewed by safety professionals.
Notification sending should be tested carefully to avoid false or accidental emergency messages.
Credentials and private configuration must never be placed in public files.
